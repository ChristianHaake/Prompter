## 5. Local Processing of User Input

Prompter processes user-entered content locally in the browser.

Entered texts are not transmitted to the operator and are not stored permanently
by the operator. The current draft is stored in the browser under the
`localStorage` key `prompter_project_v1` so it can be restored in the same
browser.

Pitch history is stored separately under the `localStorage` key
`prompter_pitch_history_v1`. It contains timestamps, target durations, actual
durations, and whether a run was completed or cancelled. This data stays in the
browser and can be cleared in the editor.

Users can delete the current draft with "Reset local data" in the app. Pitch
history has its own clear action. All local site data can also be removed
through the browser's site-data settings. Exported `.prompter` files are
generated locally by the browser and then exist outside the app on the user's
file system.
